¿Qué es un Agente? 🤖

Un agente es una entidad capaz de percibir, razonar y actuar.

Un sistema es la composición de múltiples agentes especializados que colaboran para alcanzar un objetivo común.

💡 Distinción Importante:

Un LLM por sí solo no es un agente.

Un LLM es únicamente el "cerebro" que procesa información y genera respuestas. Se convierte en un agente cuando puede percibir el entorno y actuar sobre él.

🎯 ¿Bob es un Agente o un Sistema?

Bob NO es un agente; Bob es un sistema que coordina agentes, modelos de IA, herramientas y servicios empresariales para ayudar al usuario a completar tareas complejas.

Skills Personalizadas 🎯

¿Qué son las Skills Personalizadas?

Flujos de trabajo determinísticos y especializados que extienden las capacidades de Bob para tareas específicas.
⚠️ Las skills se activan únicamente en modo Agent.

⚙️

Características

  • Flujos de trabajo predefinidos
  • Ejecución determinística
  • Pasos automatizados
  • Resultados consistentes
🔧

Casos de Uso

  • Pipelines de CI/CD
  • Migraciones de base de datos
  • Generación de reportes
  • Validación de código
📦

Beneficios

  • Automatización de tareas repetitivas
  • Reducción de errores humanos
  • Estandarización de procesos
  • Ahorro de tiempo
🚀

Integración

  • Acceso vía Modo Agente
  • Configuración personalizada
  • Reutilizable entre proyectos
  • Fácil mantenimiento

Ingeniería de Prompts ✍️

¿Qué es la Ingeniería de Prompts?

El arte y la ciencia de crear instrucciones efectivas para obtener los mejores resultados de los asistentes de IA.

🎯

Sé Específico

Proporciona requisitos claros y detallados

❌ "Haz un sitio web"

✅ "Crea una página de inicio responsiva con HTML/CSS que incluya una sección hero, barra de navegación y formulario de contacto"
📋

Proporciona Contexto

Incluye información de fondo relevante

✅ "Estoy construyendo una aplicación React para e-commerce. Crea un componente de tarjeta de producto que muestre imagen, título, precio y botón de agregar al carrito"

Personalización Avanzada de Bob 🎨

Estructura de Carpetas de Configuración

Bob utiliza el directorio ~/.bob/ para almacenar configuraciones personalizadas que modifican su comportamiento.

📋

~/.bob/rules/

Custom Rules - Instrucciones Globales

  • Archivos Markdown (*.md)
  • Se inyectan en el system prompt
  • Aplican a todos los modos
  • Ejemplo: concise-completion-rule.md
# my-rule.md ## Objective Brief description ## Rules 1. Specific instruction 2. Specific instruction
⚙️

~/.bob/settings/

Configuraciones Generales

  • custom_modes.yaml - Define modos personalizados
  • mcp_settings.json - Configuración de MCPs
  • Configuración centralizada de Bob
# custom_modes.yaml modes: - name: "Custom Mode" slug: "custom" description: "..."
⚡

~/.bob/skills/

Skills Especializadas

  • Carpetas con skills personalizadas
  • Archivo SKILL.md define la skill
  • Capacidades reutilizables
  • Ejemplo: vulnerabilities/SKILL.md
skills/ └── vulnerabilities/ └── SKILL.md

💡 Estructura Real de ~/.bob/

.bob/ ├── rules/ │ └── concise-completion-rule.md ├── settings/ │ ├── custom_modes.yaml │ └── mcp_settings.json └── skills/ └── vulnerabilities/ └── SKILL.md

📌 Nota: Los modos se definen en custom_modes.yaml, las skills en carpetas dentro de skills/ con un archivo SKILL.md, y las rules son archivos .md en rules/

Conclusión y Preguntas 🎉

Puntos Clave

  • Los asistentes de IA como Bob son herramientas poderosas para desarrolladores
  • La ingeniería de prompts efectiva es crucial para mejores resultados
  • Usa la IA como acelerador de aprendizaje, no como reemplazo del aprendizaje
  • Tú eres el responsable final del código generado por IA - siempre revisa, prueba y entiende el código generado; asume la responsabilidad de validar, entender y mantener todo el código que uses
  • AI SDLC (Software Development Life Cicle) la IA no reemplaza el ciclo de desarrollo, sino que lo transforma.
